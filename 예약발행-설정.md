# 예약 발행 — 자동 공개 설정 (한 번만 설정하면 끝)

예약 발행한 글이 **지정한 시각에 자동으로 뜨게** 하려면, 그 시각에 사이트를
한 번 다시 빌드해줘야 합니다. 무료로 매시간 자동 재빌드를 걸어두는 방법입니다.

> 글에 예약 시각을 넣으면(=`publishAt`), 그 시각 전까지는 빌드해도 사이트에
> 보이지 않다가, 시각이 지난 뒤 재빌드되면 자동으로 공개됩니다.

## 1단계 — Vercel "Deploy Hook" 만들기

1. Vercel → 프로젝트(samujangs-blog) → **Settings → Git → Deploy Hooks**
2. 이름: `scheduled-publish`, 브랜치: `main` 입력 후 **Create Hook**
3. 생성된 **URL을 복사** (형태: `https://api.vercel.com/v1/integrations/deploy/...`)
   - 이 URL로 요청을 보내면 사이트가 다시 빌드됩니다.

## 2단계 — 무료 크론으로 매시간 호출 (cron-job.org)

1. [cron-job.org](https://cron-job.org) 가입 (무료)
2. **Create cronjob**
   - **URL**: 1단계에서 복사한 Deploy Hook URL 붙여넣기
   - **Schedule**: `Every hour`(매시간) 또는 원하는 주기
   - **Request method**: `POST`
3. 저장 → 끝!

이제 매시간 사이트가 자동 재빌드되어, 예약 시각이 지난 글이 그 시간대에 공개됩니다.

## 사용 방법 (글쓰기)

1. `/admin`에서 글 작성
2. 우측 설정에서 **⏰ 예약 발행** 체크 → 날짜·시간 선택 (한국 시간 기준)
3. **저장 & 발행** → 글은 저장되지만 사이트엔 숨겨져 있다가, 지정 시각이 지나면
   다음 재빌드 때 자동 공개

> 참고: cron 주기가 "매시간"이면 공개 시각 오차는 최대 1시간 이내입니다.
> 더 촘촘히 원하면 cron 주기를 짧게(예: 30분) 잡으면 됩니다.
